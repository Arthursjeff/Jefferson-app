# Jefferson App - Arquitetura

## Objetivo principal

Criar uma plataforma interna modular para centralizar processos da Jefferson Brasil.

## Objetivo da primeira validação

A primeira versão do sistema terá foco principal na Fila de Pedidos.

O objetivo é apresentar aos supervisores uma versão funcional da fila, com controle dos pedidos, setores, estágios e permissões básicas.

## Estratégia do projeto

O sistema será estruturado em módulos.

Mesmo que a primeira entrega seja focada na Fila de Pedidos, a estrutura deverá permitir a inclusão futura de novos módulos, como orçamentos, clientes, produtos, estoque ou outros processos internos.

## Módulos iniciais

### 1. Dashboard

Página inicial do sistema.

Na primeira versão, poderá conter apenas atalhos e resumo da fila.

### 2. Fila de Pedidos

Módulo principal da primeira versão.

Responsável por controlar o avanço dos pedidos entre os estágios operacionais.

### 3. Administração

Módulo para controle de usuários, setores, permissões e configurações básicas.

### 4. Módulos futuros

Espaço reservado para futuras funcionalidades.

Exemplos possíveis:
- Orçamentos
- Clientes
- Produtos
- Estoque
- Indicadores
- Relatórios

Esses módulos ainda não fazem parte da validação inicial.

## Prioridade atual

1. Estruturar o projeto em módulos.
2. Garantir que a Fila de Pedidos funcione corretamente.
3. Publicar a versão funcional no Streamlit Cloud.
4. Apresentar a validação aos supervisores.
5. Só depois expandir para novos módulos.
